# Bug Search — 搜尋過往 Bug 解法

搜尋 Notion 中「任務追蹤工具」和「Bug 知識庫」資料庫，找出過往類似的 Bug 及其解法。

---

## 設定檔

執行前依序檢查以下路徑，讀取第一個找到的設定檔：
1. `~/.claude-company/bug-workflow-config.md`（公司環境）
2. `~/.claude/bug-workflow-config.md`（個人環境）

若都不存在，提示使用者先執行 `/bug-setup`。

---

## 流程

> **前置檢查**：參照 `references/prerequisites.md` 檢查設定檔是否存在。

### 1. 解析搜尋關鍵字

使用者輸入格式：

```
/bug-search <關鍵字>
```

例如：
- `/bug-search SSO 登入`
- `/bug-search 推播失敗`
- `/bug-search Rich Menu 排程`

### 2. 搜尋 Notion

使用 `notion-search` 同時搜尋以下來源：

**搜尋 1**：以關鍵字搜尋整個 Workspace，聚焦 bug 相關內容

```
query: <使用者關鍵字>
```

### 3. 篩選與排序結果

從搜尋結果中篩選：
- 來自「任務追蹤工具」的條目（任務類型含「🐞 錯誤」）
- 來自「Bug 知識庫」的條目
- 來自各專案下的相關 Bug 資料庫條目

排序規則：
1. 標題匹配度（關鍵字出現在標題中）
2. 時間（最近的排前面）

### 4. 顯示結果摘要

以表格形式顯示前 10 筆結果：

```
找到 N 筆相關 Bug

| # | 標題 | 專案 | 日期 | 來源 |
|---|------|------|------|------|
| 1 | SSO登入找不到使用者 | 北市府 | 2025-12-16 | 任務追蹤 |
| 2 | SSO Token 過期處理 | 北市府 | 2025-10-03 | 知識庫 |
| ...

輸入編號可查看完整內容，或輸入 0 結束搜尋。
```

### 5. 展開詳細內容

使用者選擇某筆後，用 `notion-fetch` 取得完整頁面內容，擷取並顯示：

- **問題描述**：從「🔴 問題描述」區塊擷取
- **根因**：從「🧠 根因分析」區塊擷取
- **解法**：從「✅ 修復方案」區塊擷取
- **經驗教訓**：從「📝 經驗教訓」區塊擷取（若有）
- **Notion 連結**：方便使用者跳轉查看完整內容

若頁面不符合標準模板格式（舊資料），則直接顯示完整頁面內容。

---

## 邊界情況

- **設定檔不存在**：提示使用者先執行 `/bug-setup` 完成初始設定
- **無搜尋結果**：建議使用者換個關鍵字，或提供常用的 Tags 分類供參考
- **搜尋結果過多**：僅顯示前 10 筆，提示使用者用更精確的關鍵字
- **舊資料格式不統一**：直接顯示原始頁面內容，不強制套用模板擷取
